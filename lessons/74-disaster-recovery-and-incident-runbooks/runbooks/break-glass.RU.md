# Runbook: Break-Glass Terraform / AWS Recovery

## Определение

Break-glass означает аварийный путь вне обычной автоматизации.

## Допустимые причины

- Production недоступен.
- CI не может получить role.
- Terraform backend недоступен.
- Обычный apply path слишком медленный для активного инцидента.
- Security control нужно изменить немедленно.

## Жёсткие правила

- Break-glass не используется для удобства.
- Доступ ограничен по времени.
- Каждое действие записывается.
- Ручные изменения должны быть reconciled обратно в Terraform.
- Post-incident review обязателен.
- Break-glass доступ должен быть отозван после инцидента.

## Обязательные доказательства

- что было сделано;
- кто это сделал;
- когда;
- почему обычного пути было недостаточно;
- точные команды или действия в console;
- как Terraform control был восстановлен;
- follow-up action.
